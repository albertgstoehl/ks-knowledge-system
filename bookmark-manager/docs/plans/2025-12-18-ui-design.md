## Overview

Minimal UI for bookmark management with two components:
1. **Telegram Bot** - Input entry point (Android share â†’ bot â†’ saved)
2. **Web UI** - Browsing and managing bookmarks

### Browsing Bookmarks (Web)

Two views:
- **Inbox** - Unread bookmarks, no search (forces processing)
- **Archive** - Read bookmarks, with search

Master-detail layout:
- Desktop: list on left, detail panel on right
- Mobile: list view â†’ tap â†’ full-screen detail view

### List Item Display

Each item shows: **Title â€” domain**

Example:
```

```

### Hover Interaction

Hover over list item:
- Light gray background
- Subtle shadow lift: `box-shadow: 3px 3px 0 #000`
- Smooth transition: `transition: all 0.15s ease`

No actions revealed on hover. Click opens detail view.

### Detail View Actions

- **Open** - Opens link in new tab
- **Edit** - Edit title (inline or modal)
- **Mark Read / Unread** - Toggle state, moves between Inbox/Archive
- **Delete** - Remove with confirmation

### Search (Archive only)

Search bar appears only in Archive view. Searches title, description, URL.

No search in Inbox - forces user to process items rather than search through clutter.

### Visual Style

Reference: [Physical Intelligence](https://www.physicalintelligence.company/)

- **Font**: Monospace (system monospace or similar)
- **Colors**: Black text, white background
- **Hover**: Light gray bg (#f5f5f5) + shadow
- **Layout**: Single column list, generous whitespace
- **No decorations**: No icons, no gradients, no rounded corners

## Technical Implementation

### New Files

```
src/
  routers/
    ui.py           # UI routes
  templates/
    base.html       # Base template
    index.html      # Main UI
static/
  style.css         # Styles
  app.js            # Minimal JS
bot/
  main.py           # Telegram bot
  Dockerfile        # Bot container
```

### New Endpoints

- `GET /` - Redirect to /ui/
- `GET /ui/` - Main UI page
- `GET /ui/inbox` - Inbox view
- `GET /ui/archive` - Archive view
- `GET /ui/bookmark/{id}` - Detail view (for mobile/direct links)

### Static Files

FastAPI serves static files from `/static/` path.

### Bot Container

Separate service in docker-compose.yml:

```yaml
telegram-bot:
  build: ./bot
  environment:
    - TELEGRAM_BOT_TOKEN=${TELEGRAM_BOT_TOKEN}
    - API_URL=http://app:8000
  depends_on:
    - app
```
